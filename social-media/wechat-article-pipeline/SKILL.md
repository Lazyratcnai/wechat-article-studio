# 公众号文章完整管线 v2.0

## 核心理念

**写+审双角色**：仿 dayu-agent 方法，写稿和审核分离。Writer 负责创作，Reviewer 独立核查事实。宁可多花 1 块钱审计，不发有硬伤的文章。

## 完整流程（9 步）

```
选题过滤 → 定角度 → 收集数据 → 写初稿 → 
L1 事实审计 → L2 传播审计 → 修改 → 封面图 → 发布
```

### Step 1: 选题过滤（5 维度）

用一个 `execute_code` 同时搜多个源：

```
中文: web_search × 2-3（AI热点、大模型动态）
英文: HN Algolia API × 3-4（Claude, OpenAI, AI, deepseek）
      curl -s "https://hn.algolia.com/api/v1/search?tags=story&hitsPerPage=8&query=KEYWORD"
```

选题评估：

| 维度 | 标准 |
|------|------|
| 时效性 | 48 小时内 |
| 争议性 | 有不同立场可碰撞 |
| 有用性 | 读者能带走什么 |
| 可操作 | 数据能多源验证 |
| 差异化 | 我能怎么写不一样 |

**2 项以上不满足就不写。**

### Step 2: 定角度（核心论点）

选题确定后，先想清楚：**这篇文章的一句话核心论点是什么？**

- ❌ 「DeepSeek V4 发布了」→ 新闻简报
- ✅ 「这次发布真正的新闻不是万亿参数，是一句示弱的话」→ 有观点

展示 2-3 个话题给用户选，每个附一句话角度。用户选定后再往下走。

### Step 3: 收集数据（多源交叉验证）

用 `web_extract` 抓 3-5 篇原文，提取具体数据点：

- 数字（价格、参数、百分比）
- 事件（时间线、决策、公告）
- 引用（研报原文、高管原话）
- 市场反应（股价、社区讨论）

**关键**：每个数据点至少 2 个独立来源确认。只有一个来源的数据，写的时候加「据报道」「据测算」。

数据源优先级：
1. 财联社、36kr、新浪财经（中文一手）
2. HN Algolia/Firebase（英文社区）
3. 官方文档/公告
4. ❌ 不用的：自媒体洗稿、SEO 站、知乎个人专栏（除非交叉验证通过）

### Step 4: 写初稿

**写作原则**：
- 正文不重复标题（frontmatter 已有 title）
- 开头 3 句内进入 hook
- 每段不超过手机屏幕 2.5 屏
- 短句为主，少用长从句
- 数据宁可模糊（「约 50%」）也不编造（「52.3%」）
- 结尾给读者一个「转发理由」（金句/行动建议/开放问题）

**叙事结构**：
```
冲突/悬念 → 展开（背景+数据） → 揭示（独到分析） → 回响（金句结尾）
```

**格式**：
```yaml
---
title: 有观点的标题
author: Mario
category: 科技
tags: [相关标签]
summary: 简短摘要（不要和正文重复）
cover: /path/to/cover.jpg
---
```

正文用 markdown，`##` 二级标题，重点加 **粗体**。

### Step 5: L1 事实审计（delegate_task）

用 `delegate_task` 启动独立审核 agent：

**审核内容**：
- 每个数字回溯原文验证
- 每个机构名、报告名称是否准确
- 时间线是否一致
- 引用是否断章取义
- 有没有编造的对话、人物、场景

**输出**：逐项标注 ✅/⚠️/❌，给出修改建议。

**成本**：v4-pro 约 ¥1-1.5/次（341K input tokens），占总成本 80%。

**关键**：给审计 agent 的 context 里必须包含原始数据源的完整内容，不能只给摘要。

### Step 6: L2 传播审计（主 agent 自查）

基于 L1 修改后的版本，检查：

| 检查项 | 标准 |
|--------|------|
| 前 3 句 | 能不能让人想继续读 |
| 段落长度 | 每段 ≤ 手机 2.5 屏 |
| 金句密度 | 有没有可截图转发的句子 |
| 结尾力 | 有没有给读者转发/收藏的理由 |
| 标题 | 有悬念 + 有态度 + 具体 |

### Step 7: 修改

根据 L1 + L2 审计结果修改。**重点修 🔴 严重问题**，🟡 中等问题视情况处理，🟢 可忽略。

### Step 8: 封面图

**当前方案**（腾讯混元开通前）：
- PIL 程序生成：深色渐变背景 + 芯片几何图案 + NotoSansCJK Black 大字排版
- Pollinations.ai 备选（简单 prompt 有时能出好图）

**计划方案**（等用户开通腾讯混元生图）：
- 混元生图 3.0：50 次免费额度
- 混元生图 2.0：50 次免费额度
- 质量远超 PIL 和 Pollinations

**封面图要求**：
- 900×500（16:9）
- 必须包含文章主题关键词
- 不用泛泛的「AI 科技风」
- 不用「深度分析 · Mario」等标注

## 已验证的效果

首篇试跑（DeepSeek V4 × 华为昇腾）审计结果：
- 🔴 2 处严重错误：性能对比夸大、参数表述不准确
- 🟡 5 处中等问题：训练/推理硬件混淆、收入占比加了限定词等
- 全部在发布前修正

**结论**：审计环节值这 1 块钱。
